# 泛型程序设计

# 引入泛型

假如我们有一个运算工具类Operates , 通过这个类的静态方法可以实现两个整数的加减法, 如下所示:

class Operates {
    public static int add(int i, int j){return i+j;}
	public static int sub(int i, int j){return i-j;}
}
1
2
3
4

很显然, 这个运算只能作用于整数类型int , 如果日后我们想要对浮点类型进行如上功能的实现, 则需要重新编写一个类:

class Operates {
    public static double add(double i, double j){return i+j;}
	public static double sub(double i, double j){return i-j;}
}
1
2
3
4

仔细观察这两个类, 似乎除了变量的类型发生了变化之外, 好像没有任何其他的变化.

如果我们采用了泛型程序设置, 则可以只编写一个类的代码, 我们不指明具体的数据类型是int 还是double , 而用一个名称代替它. 而当别人想用这些功能时, 只需自己提供需要的类型便可:

// 用T表示类型
class Operates<T> {
    public static T add(T i, T j){return i+j;}
	public static T sub(T i, T j){return i-j;}
}
// 使用时: (T会自动被替换成int)
Operates<int> op = new Operates();
int result = op.add(1, 2);

# 在集合中使用泛型

假如没有泛型, 则假如现在要求保存一组学生的成绩, 有如下代码:

/**
* 在集合中使用泛型之前的情况
*/
@Test
public void test1() {
    // 需求: 存放学生的成绩
    ArrayList list = new ArrayList();
    list.add(75);
    list.add(80);
    list.add(99);


    // 问题一: 类型不安全
    // 编译器不会报错, 执行也不会报错
    // list.add("Tom");

    for (Object score : list) {
        // 问题二: 强转时, 可能出现ClassCastException
        int stuScore = (Integer) score;
        System.out.println(stuScore);
    }
}

该代码有两个问题, 如上面的注释所示, 主要是类型方面的错误, 而如果采用泛型机制, 则可以避免这种问题:

/**
* 在集合中使用泛型之后的情况
*/
@Test
public void test2() {
    // 需求: 存放学生的成绩
    // 后面的Integer可以省略不写
    // <E>必须是非基本数据类型
    ArrayList<Integer> list = new ArrayList<>();
    list.add(75);
    list.add(80);
    list.add(99);


    // 如果类型出错, 无法通过编译
    // list.add("Tom");


    // 输出一
    for (Integer integer : list) {
        System.out.println(integer);
    }
    // 输出二
    Iterator<Integer> iterator = list.iterator();
    while(iterator.hasNext()) {
        Integer next = iterator.next();
        System.out.println(iterator);
    }
}

# 在比较器中使用泛型

String[] arr = new String[]{"BB", "CC", "AA", "DD"};
Arrays.sort(arr, new Comparator() {
    @Override
    public int compare(Object o1, Object o2) {
        if (o1 instanceof String && o2 instanceof String) {
            String s1 = (String) o1;
            String s2 = (String) o2;
            return -s1.compareTo(s2);
        }
        throw new RuntimeException("Type Error");
    }
})

# 在自定方法中使用泛型

# 泛型说明

  • 泛型如果不指定, 将被擦除, 泛型对应的类型均按照Object处理, 但不等价于Object

  • 经验: 泛型要使用就一路都用. 要不用, 就一路都不用

  • 静态方法中不能使用泛型, 这是因为静态方法早于类被加载于内存中

    public class Utils<T> {
    // 不可以
    public static void show() {
        //...
    }
}

# 反射机制

# 什么是反射

我们之前是怎么创建一个类的实例的?

String s = new String();
1

而反射是另一种创建对象的方式.

# 实现反射的三种方式

假如我们有一个包com.people , 在这个包中有许多不同的类: Employee , Manager :

# 第一种方法

// 实现反射的方式一:
Class c = Class.forName("com.people.Employee");
1
2

第一种方式就是调用Class类的静态方法forName , 这个方法的参数是一个类名**(必须包含完整的包名)**, 它返回一个Class 类实例, 我们可以通过这个类实例来创建它对应类类型的对象:

Class c = Class.forName("com.people.Employee");
// 通过newInstance方法创建了一个实例
Employee e = c.newInstance();
1
2
3

你的代码工具可能会提醒你newInstance 方法已经被弃用了, 关于这个将在以后提到.

# 第二种方法

Object 对象有一个方法getClass() , 这个方法可以返回一个Class 类实例, 也就是说, 假如我们想获取Employee 类对应的Class 实例, 只需用一个Employee 类的实例调用这个方法就可以了:

Employee e = new Employee();
Class c = e.getClass();
1
2

# 第三种方法

第三种方法和第二种方法类似, 同样对一个类实例进行操作:

Employee e = new Employee();
Class c = e.Class;
1
2

# 数据和代码分离

# 为什么要用反射

我们可以看到, 上面用反射创建实例的例子中:

String className = "com.people.Employee";
Class c = Class.forName(className);
Object e = c.newInstance();
1
2
3

我们只用修改className这个字符串, 就可以改变之后生成的e的类型了. 现在假如我们把这个类型的名字保存在一个外部文件中, 而在程序中通过某种方式读取这个字符串, 那是不是意味着, 即使我们的代码已经固定了, 或者已经编译好了, 那么只要修改这个外部文件, 就可以修改所生成的类了呢?

我们在src目录下创建一个test.properties 文件, 文件内容如下:

className=com.people.Employee
1

同时, 修改我们的Java代码:

FileReader reader = new FileReader("test.properties");
Properties p = new Properties();
p.load(reader);
reader.close();
String className = p.getProperty("className");
1
2
3
4
5

这样当我们下次想换成创建Manager 类实例时, 不需要重新编译或修改源代码, 只需修改配置文件就可以了.

反射使得代码更加灵活.

之前我们创建类实例的方式很死板, 我们的源代码是写死的. 这也就是说, 我们编译后, 每次运行到这里, Java虚拟机闭着眼都知道创建的肯定是这个类的实例.

而通过反射, 则可以"动态地"选择程序在运行到某个地方时要实例化的是哪种类:

反射机制在许多框架中都有使用, 我们只需要修改外部的配置文件, 就可以将大型框架跑起来, 在外部配置文件里保存的一些内容可能就是反射要用到的数据. 之后学习到的JDBC中就有用到反射机制.

getResource()  // 获取资源, 当前线程的类加载器默认从类的根路径下加载资源
