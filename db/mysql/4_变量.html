# 系统变量

系统变量是由系统定义的, 属于服务器层面.

系统变量中, 全局变量可以跨连接. 就是说, 假如我将一个全局变量G 的值从1 修改成0 后, 在别的地方连接上数据库时G 的值仍然是0 . 但是,**全局变量不能跨重启. **也就是说, 当数据库重启后, G 的值会恢复为其默认值1.

如何查看系统变量了呢? 看如下几个例子和他们的作用:

--查看所有会话变量
SHOW VARIABLES;
SHOW SESSION VARIABLES;
--查看所有全局变量
SHOW GLOBAL VARIABLES;

# 自定义变量 用户变量

使用自定义变量的步骤:

  1. 声明

  2. 赋值

  3. 使用(查看, 比较, 运算)

用户变量作用域: 针对于当前会话有效, 同于会话变量的作用域

声明并初始化有如下三种方式推荐:

SET @var_name = val;
SET @var_name := val;
SELECT @var_name := val;
1
2
3

赋值的方式可以通过上述三种方式, 切这样做可以不管数据的类型, 如:

SET @val = '张三';
SET @val = 100;

2

除此之外, 还可以用这样的方式:

-- 将学生的数量赋值给count
SELECT COUNT(*) INTO @count
FROM student;

# 函数

函数是一组预先编译好的SQL语句集合, 可以提高代码的复用性, 简化操作, 提高效率

函数只能有一个返回值.

# 创建函数

CREATE FUNCTION 函数名(参数列表)
RETURNS 返回类型
BEGIN
	函数体
END
1
2
3
4
5

  • 函数体必须要有RETURN 语句, 否则会报错.RETURN 可以不不在函数体的最后一句, 但是不建议这么做

  • 函数体如果只有一句, 可以省略BEGIN 和END

  • 使用DELIMETER 语句设置结束标记

# 调用函数

SELECT 函数名(参数列表);

案例

-- 创建一个函数, 可以返回学号对应学生的姓名
CREATE FUNCTION getInfo(id INT) RETURNS VARCHAR(8)
BEGIN
	DECLARE tmp VARCHAR(8);
	SELECT s_name INTO tmp
	FROM student WHERE s_id = id;
	RETURN tmp;
END

# 存储过程

存储过程相当于将一系列操作封装起来

# 创建语法

CREATE PROCEDURE 存储过程名称(参数列表)
BEGIN
	存储过程体
END
1
2
3
4

参数列表包括三个部分:

  • 参数模式

    • IN: 该参数可以作为输入, 也就是需要调用者传入值

    • OUT: 该参数可以返回值

    • INOUT: 该参数可以传入和返回

  • 参数名

  • 参数类型

如果存储过程体仅有一句话, BEGIN 和END 可以省略

另外, 在存储过程体中, 由于有很多其他的查询语句, 所以MYSQL不知道哪里是存储过程地结尾, 所以需要用DELIMITER 重新设置(这个过程只能在MySQL的黑窗口使用)

# 调用语法

CALL 存储过程名(参数列表);
