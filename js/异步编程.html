# 执行顺序的问题

JavaScript是单线程的, 他有一个任务队列, 当空闲的时候会从任务队列中取出并执行

假如在<script>标签中有如下代码:

setTimeout(() => {
  console.log("该我上场表演了");
}, 10)
for(int i = 0; i < 2000; ++i) {
  console.log(i);
}

这个程序并不能保证"该我上场表演了"的输出会在10ms后输出, 因为: 10ms的延迟过后, 输出这句话的任务被加入了这个任务队列, 但是, 由于程序还在执行for循环, 那么, 程序并不会从任务队列中取出console.log("该我上场表演了")的命令, 而是会在打印完1999后输出:

而Promise的then则不同, 他有一个微队列的概念, 相对而言, 我们把原来的那个任务队列为宏队列, 前者具有更高的执行等级(也就是说, 先扫描微队列, 再扫描宏队列, 而不是打断宏队列)

setTimeout(() => {
  console.log("该我上场表演了");
}, 1000);

new Promise(resolve => {
  resolve();
  console.log("Promise");
}).then(value => {
  console.log("Then");
})

for (let i = 0; i < 200; ++i) {
  console.log(i);
}
